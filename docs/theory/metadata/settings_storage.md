**Объект метаданных для создания собственных механизмов хранения настроек**, когда стандартного `ХранилищеОбщихНастроек` недостаточно.

### Ключевые свойства

| Понятие | Описание | Важно |
|---|---|---|
| **Назначение** | Собственные механизмы хранения настроек | Когда стандартного недостаточно |
| **Хранение** | В объектах конфигурации, не в системных таблицах | Справочники, регистры сведений |
| **Контроль** | Полный контроль над форматом и алгоритмами | Собственная реализация |

### Практическое применение

#### Когда использовать (примеры):
1. **Особая структура хранения** — не подходит ключ-значение

### Критические нюансы

#### Стандартное vs Кастомное:
| Аспект | Стандартное (`ХранилищеОбщихНастроек`) | Хранилища настроек (метаданные) |
|---|---|---|
| **Хранение** | Системные таблицы платформы | Объекты конфигурации |
| **Структура** | Ключ-значение | Любая (разработчик определяет) |
| **Алгоритмы** | Готовые методы платформы | Собственная реализация |
| **Использование** | По умолчанию | Когда стандартного недостаточно |

### Рекомендации

#### Использовать хранилища настроек, когда:
- ✅ Требуется сложная структура данных

#### Не использовать, когда:
- ❌ Достаточно стандартного `ХранилищеОбщихНастроек`
- ❌ Простая структура ключ-значение
