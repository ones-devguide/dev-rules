**Механизм автоматического выполнения задач по расписанию или в фоновом режиме.** Регламентные задания — шаблоны, фоновые задания — конкретные выполнения.

### Ключевые свойства

| Понятие | Описание | Важно |
|---|---|---|
| **Регламентное задание** | Шаблон/настройка выполнения | Определяет ЧТО, КОГДА и КАК выполнять |
| **Фоновое задание** | Конкретный экземпляр выполнения | Создается по расписанию регламентного задания |
| **Ключ выполнения** | Контроль параллелизма | Задания с одним ключом не выполняются одновременно |

### Практическое применение

#### Типовые сценарии:
1. **Регламентные операции** — закрытие месяца, переоценка
2. **Интеграция** — выгрузка в другие системы
3. **Очистка** — удаление старых данных, архивация
4. **Расчеты** — сложные фоновые вычисления

### Ответы на ключевые вопросы

#### 1. Отладка фоновых заданий:
- Для файловых баз ключ РежимОтладки в параметрах запуска
- Включить подключение к фоновым заданиям в конфигураторе

#### 2. Предопределенные регламентные задания:
1. Создается экземпляр в базе сразу при обновлении
2. Нельзя удалить из предприятия

### Рекомендации по использованию

#### Когда использовать регламентные задания:
- ✅ Регулярные операции (ежедневно, еженедельно)
- ✅ Длительные расчеты (ночью)
- ✅ Интеграция с внешними системами

#### Когда использовать фоновые задания:
- ✅ Отложенная обработка по запросу пользователя
- ✅ Длительные операции, которые можно запустить и забыть
- ✅ Параллельная обработка больших объемов данных
